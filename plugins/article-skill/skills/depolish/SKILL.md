# 去 AI 化润色 (De-AI Polishing)

> 让 AI 生成的中文文本读起来像真人手笔：去机械连接词、打破模板结构、增加节奏感、保留作者声音。

## When to Use

触发场景：
- **"帮我把这段去 AI 化"**
- **"这段读起来太像 AI 了，润色一下"**
- **"让这篇文章更像人写的"**
- **"去 AI 味"** / **"人工化"** / **"自然化"**
- 用户粘贴 AI 生成的中文文本并要求润色

**When NOT to use:**
- 英文文本去 AI 化（本 skill 聚焦中文）
- 用户要求重写内容 / 改变观点（这是改写，不是润色）
- 用户要求加强专业性 / 书面化（方向相反）
- 代码 / 结构化数据 / 表格

## 核心润色原则

### 1. 语言风格：自然流畅有节奏感
- ✅ 长短句交错
- ✅ 适度口语化（"说起来"、"实际上"）
- ✅ 模拟真人思考的轻微冗余
- ❌ 避免："首先 / 其次 / 最后"、"总之"、"综上所述"、"应该注意的是"、"毫无疑问"、"一般来说"

### 2. 语态调整：主动优先
- "我发现" > "被发现"
- "这个工具让我" > "我被这个工具"

### 3. 细节丰富度：增加人性化元素
- ✅ 个人见解 / 场景化描述 / 具象比喻（避免老套）
- ✅ 适度冗余：限定词（"在多数情况下"、"或许"）
- ❌ 避免绝对化断言，保留适度不确定性

### 4. 结构调整：打破模板
- ❌ 避免明显的"总分总"、"并列式"
- ✅ 自然过渡 + 偶尔的反问 / 设问

### 5. 词汇选择：具体生动
- ✅ 具体动词和名词
- ❌ 减少抽象词汇（"赋能 / 生态 / 底层逻辑"等泛词）

## 工作流

```
Step 1: 评估原文
  ├── 标记 AI 特征词（"首先 / 其次 / 总之 / 毫无疑问 ..."）
  ├── 识别模板结构（总分总 / 三段并列 / 机械列表）
  ├── 评估节奏（是否全是长句或短句）
  └── 评估情感温度（是否过度中立）

Step 2: 润色（保留核心观点）
  ├── 替换连接词：用语气词 / 过渡句替代机械连接
  ├── 打破结构：合并 / 拆分 / 重排段落
  ├── 调节奏：长句拆短 / 短句连贯
  ├── 添细节：加入限定词 / 场景例子
  ├── 换词汇：抽象 → 具体
  └── 被动 → 主动

Step 3: 对照检查
  ├── AI 高频词是否清除？
  ├── 模板感是否打破？
  ├── 核心观点是否保留？
  ├── 风格是否符合作者配置（如有）？
  └── 是否引入了事实错误？
```

## 与作者配置的结合

如果目录中有 `author-config.md` / `作者配置.md`，润色时遵循其中的：
- 人设标签（亲和力强 / 专业严谨 / 幽默风趣 / 极简干货）→ 决定口语化程度
- 代表文章语感 → 对齐句式节奏
- 常用词 / 禁用词清单

无配置时默认走"自然流畅 + 中性亲和"的通用风格。

## 与其他 skill 的协作

```
article-write   →   article-depolish
  (正文创作)         (成稿后去 AI 化兜底)
```

通常作为 `article-write` 流程的最后一步；也可独立使用于任何 AI 生成文本。

## 关键约束

1. **不改变观点**：润色 ≠ 改写，作者的核心判断和事实必须保留
2. **不引入错误**：为了"像人"而虚构细节是不可接受的
3. **保持信息密度**：去冗词，但不要稀释内容价值
4. **先问再动**：对超过 2000 字的长稿，建议先分段确认风格方向
