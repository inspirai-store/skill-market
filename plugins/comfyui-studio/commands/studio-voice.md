# /studio-voice

语音合成与唇形同步，为视频角色生成配音并同步口型动画。

## 执行流程

### 1. 前置检查

检查 `.comfyui-studio/inventory.json` 是否存在。如果不存在，提示用户：

```
未找到环境配置文件。请先运行 /studio-setup 完成首次配置。
```

读取 inventory.json，提取 `hardware.recommended_tier`、`installed_models`、`installed_nodes`。

### 2. 加载知识

读取以下 skill 文档：

- `skills/voice-lipsync/SKILL.md` — 语音合成与唇形同步方案选择和流水线指南
- `skills/voice-lipsync/references/tool-matrix.md` — 工具对比矩阵

**严格遵循 Skill 中定义的方案和决策树**，不要使用 Skill 之外的方案。

### 3. 分析用户需求

解析用户输入，判断需求类型：

**情况 A：纯语音合成**
- 用户提供了文本内容，需要生成语音
- 输入：文本字符串
- 输出：WAV/MP3 音频文件

**情况 B：语音 + 唇形同步**
- 用户提供了文本和视频/图片，需要生成配音并同步口型
- 输入：文本 + 视频/图片路径
- 输出：带有口型同步的视频

**情况 C：已有音频的唇形同步**
- 用户提供了现成的音频文件和视频/图片
- 输入：音频路径 + 视频/图片路径
- 输出：带有口型同步的视频

### 4. 方案选择

按照 `voice-lipsync/SKILL.md` 中的决策树选择方案：

- **语音合成**：根据 Skill 中的方案选择逻辑（F5-TTS / ChatTTS / GPT-SoVITS），结合已安装节点和显存档位推荐
- **唇形同步**：根据 Skill 中的方案选择逻辑（LatentSync / Wav2Lip / SadTalker），结合显存档位推荐

参考 Skill 中定义的三条流水线：
- 快速线（低显存）
- 质量线（中高显存）
- 高级线（语音克隆场景）

### 5. 执行

根据选定的方案执行：

1. **语音合成**：按 Skill 指导构建 TTS 工作流或执行命令行工具
2. **唇形同步**（如需要）：
   - 准备音频（上一步生成或用户提供）
   - 准备视频/图片
   - 构建唇形同步工作流
   - 使用 Comfy Pilot 的 `run` 执行
3. 查看输出结果

### 6. 输出和后续

```
语音/唇形同步已完成。

输出文件：
  音频：{音频路径}（如有）
  视频：{视频路径}（如有）

后续操作：
- 调整语速/语调后重新合成
- 使用视频组装功能与其他片段拼接
- 添加字幕（使用视频组装功能）
- 混入背景音乐
```
